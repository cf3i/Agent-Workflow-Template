# Wisdom

> Reusable patterns validated as effective across multiple issues.
>
> Record "why this approach works", not "what was done" (that belongs in progress.md).

## Maintenance Rules

1. Write only during Stage 5 (Reflection); appending in other Stages is forbidden.
2. Every entry must cite at least one `issue_id` as the validation source.
3. Write only "in scenario X, Y is effective" — vague "recommendations" or "best practices" are forbidden.
4. The counter-example field (when not applicable) is required — patterns without boundaries get over-generalized.
5. Append only; modifying historical entries is forbidden.

## Entry Template

```markdown
## W-00X Title

- Source: <issue_id> / PR link
- Applicable scenario:
- Pattern description:
- Counter-example (when not applicable):
```
