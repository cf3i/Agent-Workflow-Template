## 安全变更规则

1. 涉及认证、权限、密钥的改动必须在 PR 中标注 `security-impact`。
2. 任何敏感值只允许通过环境注入，不得硬编码。
3. 发现泄漏后先轮转密钥，再修代码与文档。
